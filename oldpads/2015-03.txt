Frage: Kann man freie Software in einer nicht freien Sprache (LabView) schreiben?
Antwort: Grundsätzlich ja, sofern die obigen vier Freiheiten zu gewährleisten sind.

Namen von Programmen in Hinblick auf Markenrechte sind von der Frage der SW Lizenz unabhängig zu betrachten.

Open Source und Freie Software sind synonyme Begriffe, aber mit unterschiedlicher "Konotation":
Freie Software betont die Freiheit aus der Sichtweise der Benutzer
Open Source betont die Überlegenheit Entwicklungsmodells ("Basar-Modell") aus der Sichtweise von Entwicklern
Open Source Definition: http://opensource.org/osd
Debian free software guidelines (DFSG) http://www.debian.org/social_contract#guidelines:
- nur freie Software in main, sonst non-free

155 verschiedene Open Source Lizenzen!  (xkcd dazu http://xkcd.com/927/)
http://www.ifross.org/lizenz-center

Copyleft: Sämtliche Änderungen und Weiterentwicklungen einer Freien Software dürfen nur unter der gleichen (oder kompatiblen) Lizenz veröffentlicht werden.
Es gibt auch Lizenzen ohne Copyleft-Prinzip. Änderungen und Weiterentwicklungen von Freier Software kann in diesem Fall auch als proprietäre Software weitergegeben werden.

Lizenzen mit ohne copyleft können in beliebige andere Lizenzen überführt werden (z.B. bei 40 BSD und weitere Lizenzen der Fall)

n.b.: Lizenzen haben Urheber! Dadurch können auch Rechte eingeklagt werden.
